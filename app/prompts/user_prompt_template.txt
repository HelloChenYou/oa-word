[待检查文本分片]
{chunk_text}

[输出要求]
- 仅输出一个 JSON 对象。
- 顶层结构固定为：
{{
  "issues": []
}}
- 若发现问题，`issues` 中每一项必须严格符合以下结构：
{{
  "severity": "P0|P1|P2",
  "category": "spelling|grammar|punctuation|style|terminology|compliance|structure|logic",
  "title": "string",
  "original_text": "string",
  "suggested_text": "string",
  "reason": "string",
  "evidence": "string",
  "confidence": 0.0,
  "source": "llm",
  "position_start": 0,
  "position_end": 0
}}

[位置字段要求]
1. `position_start` 和 `position_end` 必须尽量填写。
2. 位置以当前“待检查文本分片”为基准，不是全文位置。
3. `position_start` 是 `original_text` 在当前分片中的 0 基起始字符下标。
4. `position_end` 是结束字符下标，采用左闭右开区间。
5. 示例：如果分片是 `请登录OA系统`，`original_text` 是 `登录`，则 `position_start` 为 1，`position_end` 为 3。
6. 如果无法准确定位，`position_start` 和 `position_end` 都返回 null。

[判定原则]
1. 只输出“确定存在”的问题；不确定则不输出。
2. `original_text` 必须可在“待检查文本分片”中定位。
3. `suggested_text` 必须是可直接替换的文本，且不改变原意。
4. `confidence` 取值范围为 0~1。
5. 无问题时必须返回：{{"issues":[]}}

[再次强调]
不要输出任何额外文字；不要输出 Markdown；只输出 JSON。
